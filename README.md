# AI-publish

생성형 AI 출판 제도 인식 조사를 준비하기 위한 연구 자료 저장소이다. 이 프로젝트는 선행연구 수집, 논문 텍스트 정리, 문헌 검토, 설문 설계, 통계분석 계획 수립을 한 곳에서 관리하는 것을 목표로 한다.

## 프로젝트 목적

이 저장소는 다음 작업을 체계적으로 정리하기 위해 사용한다.

1. AI 출판, AI 활용 윤리, 표시 의무, 제도 필요성과 관련된 선행연구 수집
2. 수집한 논문의 본문 텍스트 추출 및 검토 메모 작성
3. 연구배경, 연구질문, 가설 정리
4. 설문 문항 설계 및 수정
5. 통계분석 계획 문서화

현재 연구의 중심 주제는 `AI가 생성하거나 AI의 도움을 받아 제작된 출판물에 대해 사람들이 어떤 제도적 관리와 표시 기준을 필요하다고 인식하는가`이다.

## 폴더 구조

### `articles/`

원문 논문 PDF를 저장하는 폴더이다. 주제적으로 유사한 논문과 방법론적으로 참고할 논문이 함께 포함되어 있다.

예시:

- AI 출판 및 AI 활용 윤리 관련 논문
- AI 사용 공개(disclosure)와 책임성 관련 논문
- 설문 + 회귀분석 구조를 참고하기 위한 방법론 유사 논문

### `reviews/`

문헌 검토 결과를 정리하는 폴더이다.

- `reviews/article_texts/`: 각 PDF에서 추출하거나 정리한 텍스트 파일
- `reviews/2026-03-13_article_collection_review.md`: 수집 논문을 주제적 유사, 기법적 유사, 비관련 문헌으로 재분류한 문서
- `reviews/2026-03-13_survey_design_review.md`: 설문 설계 관련 검토 문서

### `design/`

연구 설계와 설문 설계를 정리하는 핵심 폴더이다.

- `2026-03-13_research_background_questions_and_survey_rationale.md`
  - 연구배경, 연구질문, 가설, 설문 각 문항군의 의미를 정리한 문서
- `2026-03-13_revised_survey_instrument.md`
  - 실제 설문 문항 초안
- `2026-03-13_analysis_hypotheses_and_plan.md`
  - 변수 구성, 가설, 기술통계, t-검정, ANOVA, 회귀분석 계획
- `2026-03-13_statistical_methods_guide.md`
  - 통계기법 참고 문서
- `2026-03-13_survey_designer_notes.md`
  - 설문 설계 메모

### 루트 문서

- `AI출판제도_설문지및분석가설설계서.docx`
  - 프로젝트 결과를 문서 형태로 정리한 파일
- `README.md`
  - 저장소 개요 및 작업 가이드

## 추천 읽기 순서

처음 이 저장소를 보는 경우 아래 순서로 읽으면 전체 흐름을 빠르게 파악할 수 있다.

1. `reviews/2026-03-13_article_collection_review.md`
2. `design/2026-03-13_research_background_questions_and_survey_rationale.md`
3. `design/2026-03-13_revised_survey_instrument.md`
4. `design/2026-03-13_analysis_hypotheses_and_plan.md`

## 현재 작업 흐름

이 프로젝트는 대체로 아래 순서로 진행된다.

1. 관련 논문 PDF를 `articles/`에 수집한다.
2. 논문 본문 텍스트를 `reviews/article_texts/`에 정리한다.
3. 문헌 검토 결과를 `reviews/`의 요약 문서에 반영한다.
4. 문헌 검토를 바탕으로 `design/`에서 설문 문항과 가설을 수정한다.
5. 통계분석 계획을 문서화하고 최종 설계서로 통합한다.

## 파일명 규칙

현재 문서들은 날짜 기반 파일명을 사용한다.

- 형식: `YYYY-MM-DD_설명.md`
- 예시: `2026-03-13_analysis_hypotheses_and_plan.md`

이 규칙을 유지하면 설계 변경 이력을 시간순으로 관리하기 쉽다.

## Git 관리 원칙

이 저장소는 연구 산출물 중심 저장소이며, 작업 보조용 에이전트 설정 폴더는 원격 저장소에 올리지 않는다.

- `.codex/`는 `.gitignore`에 포함되어 있으며 GitHub 업로드 대상에서 제외한다.
- 연구 문서, 설문 설계 문서, 검토 메모, 논문 목록만 버전 관리한다.
- PDF 원문을 계속 저장소에 포함할지 여부는 저장소 용량과 공유 범위를 고려해 별도로 판단한다.

## 향후 확장 가능 항목

필요하면 아래 항목을 추가할 수 있다.

- 설문 응답 데이터 원본 저장용 `data/`
- 분석 코드 저장용 `analysis/`
- 결과표 및 시각화 저장용 `outputs/`
- 투고용 초안 저장용 `manuscript/`

## 비고

이 저장소는 소프트웨어 패키지 프로젝트라기보다 연구 설계 및 문헌정리 저장소에 가깝다. 따라서 실행 방법보다는 문서 구조, 연구 흐름, 버전 관리 원칙을 중심으로 관리한다.
